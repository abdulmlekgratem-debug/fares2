## 2) إصلاح وضع التتبع المباشر على الجوال

### المشكلات الحالية (من `src/components/map/LiveTrackingMode.tsx`):

1. **الزوم لا يُفعَّل تلقائياً عند بدء التتبع**: حالياً السطر 342–344 يستدعي `onZoomToLocation(lat, lng, 17)` فقط إذا كان `autoZoomOut` معطلاً، لكن يستخدم زوم ثابت 17 ولا يُفرض زوم أولي قوي عند الضغط على زر "بدء التتبع".
2. **لا يحترم زوم المستخدم**: في كل تحديث موقع (السطر 343)، يُعاد ضبط الزوم إلى 17 — مما يُلغي أي زوم خارجي قام به المستخدم. النتيجة: المستخدم يحاول التكبير/التصغير فيرتد فوراً.
3. `autoZoomOut` **يتعارض مع تفاعل المستخدم**: السطور 267–280 تفرض زوم 15 أو 17 تلقائياً حتى لو كان المستخدم قد ضبط زوماً مخصصاً.
4. **الأخطاء المتقطعة على الجوال**: `maximumAge: 0` (السطر 360) يطلب تحديثاً جديداً كل مرة → استنزاف بطارية + timeouts على شبكات بطيئة. خاصية `distanceFilter` (السطر 361) ليست جزءاً من معايير `PositionOptions` → يتم تجاهلها (لا تأثير لها).
5. **لا توجد آلية fallback للجوال**: عند فشل `watchPosition` (شائع على iOS Safari)، يُغلق التتبع بدلاً من المحاولة مرة أخرى.

### الحلول التقنية المقترحة:

#### أ. منطق الزوم الجديد (السلوك المطلوب من المستخدم):

- **عند الضغط على "بدء التتبع"**: تكبير قوي تلقائي إلى زوم 18 (street level) **مرة واحدة فقط**.
- **بعدها**: المستخدم حر في التكبير/التصغير، والخريطة تتبع موقعه فقط (`panTo`) **دون لمس الزوم**.
- **عند ترك المستخدم زوماً معيناً**: التتبع يستمر بنفس الزوم الذي تركه.

التنفيذ في `LiveTrackingMode.tsx`:

```typescript
// إضافة ref لتتبع أول مرة فقط
const initialZoomDoneRef = useRef(false)

// في startTracking — تكبير أولي مرة واحدة
const startTracking = useCallback(() => {
  initialZoomDoneRef.current = false  // إعادة الضبط عند البدء
  // ... existing watchPosition setup
})

// داخل watchPosition callback — استبدال السطر 342–344:
if (!initialZoomDoneRef.current) {
  onZoomToLocation(latitude, longitude, 18)  // زوم قوي مرة واحدة
  initialZoomDoneRef.current = true
} else {
  // فقط pan دون تغيير زوم — يحترم زوم المستخدم
  onPanToLocation(latitude, longitude)
}
```

#### ب. إضافة prop جديد `onPanToLocation` يقوم بالتمركز فقط دون تغيير زوم:

في `InteractiveMap.tsx` (بجانب `handleZoomToLocation` السطر 548):

```typescript
const handlePanToLocation = useCallback((lat: number, lng: number) => {
  if (mapProvider === 'google' && map) {
    map.panTo({ lat, lng })  // بدون setZoom
  } else if (mapProvider === 'openstreetmap' && leafletMap) {
    leafletMap.panTo([lat, lng], { animate: true })
  }
}, [map, leafletMap, mapProvider])
```

#### ج. إلغاء/تعطيل `autoZoomOut` التلقائي:

- إزالة منطق فرض زوم 15/17 (السطور 266–280) لأنه يتعارض مع حرية المستخدم.
- **بديل**: زر يدوي "إعادة التمركز" فقط (موجود بالفعل `centerOnLocation` السطر 447 — تركه كما هو).

#### د. إصلاح خيارات `watchPosition` للجوال:

```typescript
{
  enableHighAccuracy: true,
  timeout: 20000,        // ← من 10000 إلى 20000 (شبكات بطيئة)
  maximumAge: 2000       // ← من 0 إلى 2000ms (يقبل موقع حديث، يوفر بطارية)
  // حذف distanceFilter — غير قياسي ومُتجاهل
}
```

#### هـ. إضافة retry تلقائي عند فشل GPS:

```typescript
(err) => {
  console.error('Geolocation error:', err)
  if (err.code === 3) {  // TIMEOUT
    setError('بطء في الإشارة، يُعاد المحاولة...')
    // إعادة تشغيل watchPosition بعد 3 ثوان
    setTimeout(() => startTracking(), 3000)
  } else if (err.code === 1) {  // PERMISSION_DENIED
    setError('يرجى السماح بالوصول للموقع من إعدادات المتصفح')
    setIsTracking(false)
  } else {
    setError('فشل تحديد الموقع. تأكد من تفعيل GPS.')
    setIsTracking(false)
  }
}
```

#### و. تحسينات إضافية للجوال:

- استخدام `wakeLock` API لمنع قفل شاشة الجوال أثناء التتبع.
- التحقق من `permissions.query({ name: 'geolocation' })` قبل البدء لإعطاء رسالة أوضح.

### الملفات المعدّلة:
